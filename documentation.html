Vue d'ensemble

Pourquoi utiliser l'assistant ?

L'outil de migration automatise la reprise des données Sage 100 vers Sage Intacct. Il procure un parcours guidé en 9 étapes, sécurise les correspondances et garde la trace de chaque action. Plus besoin de manipuler des fichiers CSV ni de lancer des scripts manuels : tout se fait dans un assistant clair, étape par étape.

Ce que l'assistant gère

  • Migre automatiquement : plan comptable, journaux, plans analytiques, contacts, clients, fournisseurs, comptes bancaires et écritures générales (avec dimensions analytiques quand les correspondances sont complètes).
  • Après migration : certains réglages restent manuels dans Intacct (TVA, workflows de validation, rapports personnalisés).
Gain de temps (x10) Migration multi-bases Relançable étape par étape Filtres avancés (période, journaux) Logs et rapports détaillés Contrôles d'équilibre automatique

Ce qu'il faut préparer

  • Accès SQL Server aux bases Sage 100 (lecture + création de tables auxiliaires).
  • Identifiants web services Sage Intacct (Company ID, Sender ID, utilisateur, mot de passe).
  • Liste des bases Sage 100 à migrer, associée aux entités (LOCATIONID) Intacct.
  • Période des écritures à migrer (par défaut : toute la période disponible ; filtres par mois et par journal en option avancée).
  • Dans Intacct, périodes comptables et séquences de pièces ouvertes pour couvrir l'ensemble des écritures à migrer.
  • Dans Intacct, séquences de numérotation configurées pour les clients et fournisseurs (les identifiants Sage 100 ne sont pas réutilisés comme codes Intacct).
  • Dans Intacct, champs personnalisés SAGE100_DB et SAGE100_ID créés sur CUSTOMER, VENDOR, GLENTRY et GLBATCH (taille recommandée : 100 caractères pour les tiers, 2000 pour les écritures).

Parcours simplifié

  1. Connexion aux environnements Sage 100 / Intacct et tests automatiques.
  2. Matching graphique des bases Sage 100 avec les entités Intacct.
  3. Lancement guidé des 9 étapes (plan comptable → écritures), relançables individuellement.
  4. Optionnel : activation des filtres avancés (période, journaux) pour l'étape Écritures — réservé aux tests ou reprise ciblée.
  5. Lecture du résumé, analyse des écarts et export des rapports de contrôle.

Préparation & configuration

Connexion SQL Server (Sage 100)

Renseignez les champs présentés dans l'écran « Connexion Sage 100 » :

  • Serveur : nom du serveur ou instance (ex. SRV-BDD\\SAGE100).
  • Base système : base contenant les dossiers (SAGE100 ou similaire).
  • Authentification : Windows ou SQL, selon vos habitudes.
  • Test de connexion : déclenche une vérification instantanée et affiche un message clair.
Bon à savoir : lors de la première connexion, l'outil crée automatiquement un espace de travail local (base ou schéma intacct) pour stocker les logs et mappings.

Connexion Sage Intacct

Renseignez les identifiants web services dédiés à la migration :

  • Company ID : identifiant de la société Intacct cible.
  • Sender ID et Sender Password : identifiants d’accès aux API Intacct, fournis lors de la création de l’application (portail développeur / partenaire).
  • User ID et User Password : utilisateur Intacct (compte web services dédié ou compte standard) avec droits suffisants pour créer les objets concernés.
Astuce : créez un utilisateur Intacct spécifique à la migration afin de tracer facilement les écritures générées.

Matching des bases Sage 100 ↔ entités Intacct

L'interface « Association des bases » présente toutes les bases détectées (ex. SOCIETE_A, SOCIETE_B). Faites glisser chaque base vers l'entité Intacct correspondante. L'outil enregistre ensuite :

  • LOCATIONID et ENTITYID utilisés lors de la création des clients/fournisseurs.
  • Informations d'entité stockées dans les champs personnalisés (ex. SAGE100_DB) pour suivre la provenance des données.
  • Couleur/nom affiché dans l'assistant pour identifier visuellement chaque base.
Attention : une base non associée n'apparaît pas dans les étapes de migration. Vérifiez que toutes les sociétés actives ont bien été matchées avant de commencer.

Assistant de migration (9 étapes)

Vue générale

Les 9 étapes suivent un ordre logique. Vous pouvez relancer une étape à tout moment ; l'outil détecte automatiquement les éléments déjà synchronisés et ignore les doublons. Chaque étape affiche :

  • Nombre total d'enregistrements détectés.
  • Progression (en attente → en cours → terminé).
  • Cartes « Erreurs » et « Ignorés » pour comprendre rapidement les anomalies.
  • À l'étape Écritures : par défaut, migration sur toute la période et tous les journaux ; les options avancées (filtre dates, choix des journaux, regroupement mensuel) sont masquées jusqu'à activation explicite.
01

Plan comptable

Harmonise les comptes généraux de toutes les bases.

  • Source : table plan comptable F_COMPTEG, comptes actifs uniquement.
  • Choisissez la base de référence : ses libellés serviront de modèle.
  • Si un compte manque dans une base secondaire, l'assistant propose un écran de mapping.
  • Résultat : comptes GL créés dans Intacct avec l'identifiant d'origine (champ personnalisé SAGE100_ID).
Conseil : vérifiez qu'aucun compte important n'est manquant ou non mappé entre les bases, afin d'éviter des erreurs lors de la migration des écritures si ces comptes sont utilisés.
02

Journaux

Crée les journaux Intacct à partir de la table F_JOURNAUX.

  • Les journaux en sommeil sont ignorés.
  • Chaque journal reprend le code Sage (ex. AC, VT).
  • Vous pouvez renommer un journal avant validation pour adopter une nomenclature commune.
03

Analytique

Relie les plans analytiques Sage 100 aux dimensions Intacct.

  • Sélectionnez un plan (ex. Axe 1) puis choisissez la dimension Intacct (Department, Location, dimension personnalisée...).
  • Plusieurs plans issus de sociétés Sage différentes peuvent être mappés vers une même dimension Intacct (ex. Services de la société A et Services de la société B → une seule dimension Services).
  • Un aperçu liste les numéros et libellés pour vérifier la cohérence avant création.
  • Les liaisons sont enregistrées afin que les écritures comptables retrouvent automatiquement la bonne dimension.
Important : mappez ici tous les plans analytiques utilisés dans les écritures à migrer ; les sections de plans non mappés ne seront pas créées comme valeurs de dimension dans Intacct et l'analytique correspondante sera perdue.
04

Contacts

Prépare les fiches contacts associées aux clients/fournisseurs.

  • La civilité Sage (Mr / Mme) est convertie automatiquement.
  • Nom + prénom sont fusionnés pour l'affichage (CONTACTNAME et PRINTAS).
  • Les numéros de téléphone, email et services sont importés depuis F_CONTACTT.
  • Les doublons sont détectés via le nom de contact déjà présent dans Intacct (CONTACTNAME).
05

Clients

Crée les comptes clients Intacct depuis F_COMPTET (CT_Type = 0).

  • Coordonnées complètes : adresses, codes postaux, pays (convertis en anglais grâce au Country Mapper intégré).
  • Association automatique avec le contact principal importé à l'étape 4.
  • Les comptes clients sont créés en utilisant les séquences Intacct ; le code Sage d'origine est mémorisé dans un champ personnalisé pour faciliter les rapprochements.
  • Les modes de règlement et limites de crédit sont repris quand ils existent.
06

Fournisseurs

Structure similaire aux clients, avec focus sur les coordonnées bancaires.

  • Jointure avec F_BANQUET pour récupérer IBAN, BIC et libellé banque.
  • Les comptes fournisseurs sont créés en utilisant les séquences Intacct ; le code Sage d'origine est mémorisé dans un champ personnalisé pour faciliter les rapprochements.
  • Association automatique aux contacts fournisseurs.
  • Les coordonnées bancaires fournisseurs sont reprises pour alimenter Intacct et faciliter la préparation des paiements.
07

Comptes bancaires

Crée les comptes bancaires avec leurs coordonnées complètes et paramètres SEPA.

  • Lecture des banques (F_BANQUE) et comptes bancaires (F_EBANQUE).
  • Import des IBAN (nettoyés) et BIC, adresse complète, devise et compte comptable associé.
  • Chaque compte est créé dans l'entité Intacct correspondante (LOCATIONID) et configuré pour les fichiers bancaires SEPA.
  • Étape indépendante des écritures comptables, recommandée pour préparer les paiements et fichiers bancaires.
08

Écritures comptables

Étape la plus volumineuse : injection des écritures générales.

  • Groupement par pièce, date et journal pour former les lots envoyés à Intacct.
  • Contrôle automatique de l'équilibre débit = crédit ; pour les journaux bancaires avec compte général de contrepartie centralisée (« toutes les écritures de la banque »), une ligne de contrepartie peut être ajoutée automatiquement.
  • Regroupement mensuel optionnel, configuré journal par journal : une écriture par mois et par journal, avec possibilité de regrouper les pièces non bancaires déséquilibrées si le mois complet est équilibré (avertissements plutôt qu'erreurs bloquantes).
  • Analytique : une seule dimension peut utiliser l'allocation personnalisée (« Custom allocation ») d'Intacct ; les autres plans mappés sont appliqués en champs directs seulement si leurs montants sont cohérents, sinon l'écriture est migrée sans la dimension concernée.
  • Les écritures déjà migrées sont détectées via le champ SAGE100_ID stocké dans la base locale.
Astuce : migrez par période (ex. mois par mois). Le filtre de période est disponible avant de lancer l'étape (sélection par mois uniquement).
Options avancées : le filtre de dates, le choix des journaux et le regroupement mensuel sont des fonctionnalités avancées, destinées aux utilisateurs avertis et aux tests. Elles sont masquées par défaut dans l'assistant ; les activer permet de restreindre la période ou certains journaux — à utiliser avec précaution.
09

Résumé & rapports

Affiche un bilan détaillé par étape.

  • Totaux créés / ignorés / en erreur (tableau récapitulatif à l'écran).
  • Les fichiers d'erreurs et de rejets sont dans le dossier logs/ (voir FAQ « Où trouver les rapports ? »).
  • Checklist finale (validation dans Intacct, blocage des écritures, rapprochement bancaire...).
  • Contrôles recommandés : comparer quelques soldes de comptes entre Sage 100 et Intacct, vérifier un ou deux journaux par mois (totaux débit/crédit), contrôler quelques écritures analytiques clés et tester la recherche par SAGE100_ID.

Gestion des erreurs & rejets

Typologie

Catégorie Symptômes Action recommandée
Connexion Tests rouges, message « Impossible de joindre le serveur » Vérifier identifiants, pare-feu, droits SQL/Intacct.
Données manquantes Comptes, journaux ou analytiques introuvables Lancer les étapes dans l'ordre ; compléter les mappings manuels proposés.
Doublons Message « déjà migré » ou « existe déjà » Laisser l'outil ignorer ces enregistrements ; vérifier le champ personnalisé dans Intacct.
Équilibre Écritures déséquilibrées, message « Pièce EC123 non équilibrée » Contrôler les montants dans Sage 100, corriger puis relancer uniquement la pièce concernée.
Logs accessibles : dossier logs/ de l'application Windows, fichiers nommés par objet (clients_erreurs.txt, journal_erreurs.txt, etc.). Chaque ligne décrit la base, l'identifiant et la raison du rejet.

Conseils d'utilisation

Avant de démarrer

  • L'accès aux bases Sage 100 est en lecture seule ; aucune modification n'est effectuée côté Sage.
  • Travailler d'abord sur une base pilote pour valider le paramétrage.
  • Préparer la liste des exceptions connues (journaux inactifs, comptes à exclure).

Modes de lancement

Arrêt sur erreur : stoppe l'étape dès qu'un rejet apparaît. Idéal lors des premiers tests.

Exécution complète : continue malgré les erreurs et les rassemble dans un rapport final.

Redémarrer une étape

  • Relancer directement depuis l'assistant : seuls les éléments restants sont traités.
  • Pour les écritures : activez les options avancées dans l'étape, puis utilisez les filtres (dates, journaux) pour cibler un batch précis.
  • Consultez la colonne « Ignorés » pour confirmer qu'une reprise partielle s'est bien passée.

FAQ & dépannage

Questions fréquentes

  • Une base n'apparaît pas dans la liste : assurez-vous qu'elle est accessible sur le serveur SQL et qu'elle contient la table P_DOSSIER. Relancez le scan des bases depuis l'écran Connexion.
  • Comment migrer uniquement les fournisseurs ? Passez directement à l'étape « Fournisseurs » ; l'outil vérifiera que les contacts sont disponibles et vous proposera de lancer la synchronisation.
  • Où trouver les rapports ? Les rapports d'erreurs et de rejets sont enregistrés dans le dossier logs/ (créé automatiquement à côté de l'application). Fichiers générés : plan_comptable_erreurs.txt, clients_erreurs.txt, fournisseurs_erreurs.txt, journal_erreurs.txt, contact_erreurs.txt, checkingAccount_erreurs.txt, journalEntry_erreurs.txt, analytique_erreurs.txt, ecritures_analytiques_ignorees.txt. L'étape Résumé affiche un tableau récapitulatif à l'écran ; il n'y a pas de bouton d'export de rapport.
  • Peut-on arrêter la migration en cours ? Oui, le bouton « Stop » met l'étape en pause. Vous pourrez reprendre plus tard sans perdre le suivi.
  • Comment modifier une entité associée ? Retournez dans l'écran de matching, modifiez l'association puis relancez les étapes concernées.
  • Comment filtrer les écritures par période ou par journal ? Dans l'étape « Écritures comptables », cliquez sur « Activer les fonctionnalités avancées » ; les filtres (dates par mois, liste des journaux, regroupement mensuel) s'affichent. Ces options sont destinées aux utilisateurs avertis et aux tests — à utiliser avec précaution.

Annexes & références

Tables SQL consultées

Structures principales

  • F_COMPTEG : plan comptable.
  • F_JOURNAUX : journaux.
  • F_COMPTET : clients (CT_Type=0) & fournisseurs (CT_Type=1).
  • F_CONTACTT : contacts.
  • F_BANQUE & F_EBANQUE : banques & IBAN.
  • F_ECRITUREC : écritures comptables.

Base locale de migration

  • ParamMigration : paramètres généraux (mode, dernières exécutions).
  • JournalMapping, GlAccountMapping : correspondances Sage ↔ Intacct.
  • AnalytiqueMapping : plans analytiques reliés aux dimensions.
  • JournalEntryLog : suivi des lots d'écritures migrés.

Liste des objets Intacct utilisés

  • GLACCOUNT : compte général (plan comptable Intacct, un enregistrement par compte).
  • JOURNAL : journal comptable (équivalent des journaux Sage 100, utilisé pour classer les écritures).
  • CONTACT : fiche contact (nom, email, téléphone) reliée aux clients et fournisseurs.
  • CUSTOMER : objet client Intacct créé à partir des fiches clients Sage 100.
  • VENDOR : objet fournisseur Intacct créé à partir des fiches fournisseurs Sage 100.
  • CHECKINGACCOUNT : compte bancaire opérationnel (utilisé pour les paiements, encaissements et rapprochements).
  • GLBATCH : lot d'écritures générales (regroupe plusieurs GLENTRY pour un envoi donné).

Chaque objet reçoit les identifiants Sage (champ SAGE100_ID) pour simplifier la réconciliation et permettre une reprise partielle sans doublon.

Exemple : à partir d'une écriture Intacct, vous pouvez lire son SAGE100_ID (base | pièce | date | journal), puis retrouver la même pièce dans Sage 100 ou dans la table locale JournalEntryLog pour analyser son historique complet.
